**No open tool can validate HSL / Color-Grade *fidelity* — only Adobe Camera Raw.** `crs:HueAdjustment*`/`crs:ColorGrade*` are Camera-Raw develop settings, NOT DNG-render params. `dng_validate` renders the DCP pipeline (stages 1-9) only; darktable / RawTherapee read a raw + *their own* sidecar params and ignore Adobe's `crs:*` develop fields entirely. So an "open cross-ref" for develop-op fidelity does not exist — ACR is the sole oracle (consistent with VALIDATION.md §a). This is also why develop_ops are orthogonal to the ship gate ([[ship-gate-render-path]]).

**The grading-sweep harness** (`tools/grading_sweep/`, `tests/test_grading_sweep.py`, PR #30) handles this in two tiers:
- Tier 0 (free, CI): synthetic linear-ProPhoto chart + full-range sweep of every HSL band / CG wheel → asserts *structural* properties (monotonicity, band/zone locality, neutral protection, no-negative, identity). The integration-level net above the Axis-1 oracle.
- Tier 1 (needs a licensed ACR): `build_sweep_xmps.py` emits Adobe-`crs` sweep sidecars; the SAME files drive both sides; the payoff is least-squares-**fitting our free constants** (band centres, `_HSL_HUE_MAX_HEX`, `_CG_CHROMA_STRENGTH`/`_CG_LUM_STRENGTH`, Blending/Balance maps) to the ACR golden set. Functional-form residual floor remains (closed-source form), like the SSF Luther floor.

**Frontier — now DECIDED + partly shipped** (`docs/research/v09-perceptual-grading-frontier.md`; DECISIONS §5/§7). The `--render-intent {faithful, perceptual}` switch is decided + scaffolded (sRGB TIFF→faithful, ACEScg EXR→perceptual; per-target default in `cli._default_intent_for_preset`).
